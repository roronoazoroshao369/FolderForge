# Support

Use GitHub Discussions when enabled, or open an issue using the appropriate
template for reproducible bugs and feature requests.

Before reporting a bug, run `folderforge doctor --json`, remove secrets from the
output, and include the FolderForge version, Node version, operating system,
installation method, command, expected result, and actual result.

Questions about unsupported environments, custom child servers, or deployment
architecture are handled on a best-effort basis. Security reports must follow
[SECURITY.md](SECURITY.md) and must not be filed publicly.
